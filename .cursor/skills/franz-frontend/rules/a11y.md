# Accessibility

Glob rule: `accessibility.mdc`. Related: `forms-and-drafts.mdc`, `responsive-design.mdc`.

## Baseline

- Semantic HTML first (`button`, `nav`, `main`, `label`)
- shadcn/Radix for dialogs, tabs, focus traps
- Keyboard: Tab, Enter/Space, Escape
- Visible focus — `ring-ring`; never `outline: none` without replacement
- **Skip link** — first focusable in root layout → `#main`

```tsx
<a href="#main" className="sr-only focus:not-sr-only focus:absolute focus:z-50 focus:p-4 focus:bg-background">
  Skip to main content
</a>
<main id="main">{children}</main>
```

## Labels

**A heading is not a label.**

```tsx
// Bad
<h2>Item name</h2>
<Input value={name} onChange={...} />

// Good
<h2 id="item-step-title">Item name</h2>
<Label htmlFor="item-name" className="sr-only">Item name</Label>
<Input id="item-name" aria-labelledby="item-step-title" ... />
```

- Every `Input` / `Textarea` / `Select` → `<Label htmlFor>` or `aria-label`
- Placeholder is not a label
- Icon-only buttons → `aria-label`; icon `aria-hidden`

## Named regions

```tsx
<TabsList aria-label="Content categories">
```

## Form errors

Connect `aria-invalid`, `aria-describedby`, and error element `id`:

```tsx
<Input
  id="title"
  aria-invalid={!!errors.title}
  aria-describedby={errors.title ? "title-error" : undefined}
/>;
{
  errors.title && (
    <p id="title-error" role="alert" className="text-sm text-destructive">
      {errors.title.message}
    </p>
  );
}
```

Server `fieldErrors` via `applyFieldErrorsToForm` must still wire these attributes.

## Live regions

- Generating / streaming-only UI → `aria-live="polite"` (or `assertive` for critical)
- `aria-busy` on sections during load when content is replaced

## Touch targets

Custom controls → `min-h-11 min-w-11` (44×44) — see `responsive-design.mdc`.

## Checklist

- [ ] Skip link + `main#main`

- [ ] Form errors use `aria-invalid` + `aria-describedby`
